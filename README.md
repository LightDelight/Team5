# MoverExampleTest

Developed with Unreal Engine 5

Mover Example을 베이스로 제작되었습니다.

Docs 폴더에 아키텍처 원칙이 정리되어 있습니다.

외곽선과 상호작용 대상이 달랐던 케이스 상세 : 아이템이 하나의 위치에서 한 번에 스폰될 때 자연스럽게 겹치게 되는데, 이때 서버와 클라이언트의 아주 약간의 미세한 차이 (아마 스폰 타이밍이나 아주 약간의 물리 오차 문제?) 로 인한 거리 기반 판정에 차이가 있었는데, 이때 TryInteract 함수를 클라이언트가 Run On Server로 호출할 경우, 서버 플레이어 캐릭터의 기준, 즉 같은 위치에 섰을 때 서버 플레이어에게 외곽선으로 잡히는 대상이 상호작용 판정 대상이 되는 버그.
클라이언트 플레이어 캐릭터와 서버 플레이어 캐릭터가 다른데도 이런 일이 일어난 이유 추측 : TryInteract는 자신이 BestTarget으로 잡은 대상을 담아 서버로 보내는 방식인데, Run On Server 이벤트로 호출할 경우 서버에 있는 플레이어 캐릭터가, 다시 말해 아이템들의 위치와 SphereCollision의 Overlap과 거리 판정을 서버의 월드 기준으로 판단했기 때문인 것으로 보임.
